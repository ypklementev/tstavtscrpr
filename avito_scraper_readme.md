# Avito Scraper (наглядная демонстрация алгоритма)

## Общее

Проект основан на Playwright и предназначен для парсинга номеров продавцов
Данный мини проект предназначен для демонстрации алгоримта работы крупного парсера
преимуществом является то, что вы можете быстро его запустить и убедиться, что парсер работает, не занимаясь настройкой прокси и системой обхода капчи

------------------------------------------------------------------------

## Алгоритм

1.  Открывает страницу
2.  Дожидается загрузки элементов (`wait_for_items`)
3.  Перебирает элементы
4.  Находит кнопку "Показать телефон" (`wait_for_phone_button`)
5.  Нажимает кнопку (`safe_click`)
6.  Ожидает изображение
7.  Извлекает base64 изображение
8.  Сохраняет как `phone_X.png`
9.  Запускает OCR для изображения
10. Пишет в TXT связку из заголовка объявления и номера телефона

------------------------------------------------------------------------

## Ключевые компоненты

### wait_for_items

Ожидает пока элементы появятся в DOM.

### wait_for_phone_button

Находит кнопку отображения телефона в каждом айтеме используя повторные попытки при неудаче.

### safe_click

Симулирует человеческое поведение при нажатии кнопки.

### wait_for_new_phone_image

Обнаруживает новые изображения с номерами.

------------------------------------------------------------------------

## Известные проблемы

### 1. CAPTCHA / IP blocks

Avito может блокировать доступ и отправлять на hcaptcha при частых запросах и еще ряду причин (капчу можно пройти вручную и парсер продолжит работу)

### 2. Производительность

Текущее решение не оптимально по скорости: 
- UI взаимодействия (иных путей пока не вижу) 
- Задержки между запросами (для избежания anti-bot защиты) 
- OCR занимает около 0.5с на номер

------------------------------------------------------------------------

## Ограничения

-   Сильная зависимость от UI автоматизации
-   Требует ручного решения CAPTCHA
-   Не масштабируем без Proxy
-   Возможные ошибки OCR (не выявлено)
